- the program take as argument a sequence of positive integer
- we need to sort it by using the merge-insert sort algorithm
- we need to use at least two different container, same algorithm but two container (to show that container matter)
- we need to mesure the execution time of each container
- we print the result

MERGE-INSERT SORT: (ex : 3 5 9 7 4)
1 - group the elements into pairs : (3, 5) (9, 7) (4)
2 - sort each pair : (3, 5) (7, 9) (4)
3 - separate into two groups : small: [3, 7, 4] big: [5, 9]
4 - sort only the big : sorted big: [5, 9] lower_bound
5 - insert the small one by one :
    insert 3 -> [3, 5, 9]
    insert 7 -> [3, 5, 7, 9]
    insert 4 -> [3, 4, 5, 7, 9]

STEP:
1- parse the argument to detect errors:
    - no argument
    - non numeric argument
    - negative argument
    - overflow
2 - if valid argument, we start the time, we built the two container
3 - we implement the same merge-insert sor algorithm for the two container
4 - we print the result with the time that take our two container to sort them
