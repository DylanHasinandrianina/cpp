Standard template library

vector : like an array but dinamically changable, can be accessed by index, can add only add or delete a the end.
list : like a vector but can add and delete from both beginning or the end of the list.
function utils:
    .at() : behave like index but more safe because if show an error if out of bound.
    .font(), .back() : access or change first and last element.
    .begin(), .end() : iteraors to the first or the last element.
    .push_front(), push_back() : add element to the list from the front or the back (N.B : can only use push_back for vector).
    .pop_front(), pop_back() : remove element to the list from the front or the back (N.B : can only use pop_back for vector).
    .size() : return the size of the list or vector.
    .empty() : return 1 is empty.




stack : last in, first out (LIFO) (pile of pancakes)
    - we can only acces the last element add in the stack.
    - .push() : add element in the stack.
    - .top() : to access or change the the top element of the stack (the last added).
    - .pop() : remove the last element that was added to the stack.
-> added at the top, remouved from the top.

queues : first in, first out (FIFO) (people standing in line)
    similar to stack but just the first one added is the top one.
-> added at the end, remouved from the top.


N.B : stack and queues are container adapter


ITERATORS:
    -> a generalized pointer for container
    -> .begin() and .end() return an iterator to the first and last element

TYPENAME:
    -> tell the compiler. this is a type
